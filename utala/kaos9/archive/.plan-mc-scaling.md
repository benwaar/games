## Key Decisions

### How many rollout levels?
- **Recommended**: 5 levels (10, 50, 100, 200, 500)
- **Rationale**: Captures early gains, mid-range scaling, and plateau
- **Optional**: Add 1000 if time permits and 500 shows no plateau

### How many games per matchup?
- **Recommended**: 100 games (50 each position for balance)
- **Rationale**: Good statistical signal, reasonable time budget
- **Alternative**: 200 games for more precision (doubles time)

### Which baselines to test against?
- **Must have**: Random (absolute skill), Heuristic (competitive)
- **Nice to have**: Head-to-head between levels (scaling curve)
- **Rationale**: Random shows improvement, Heuristic shows ceiling

### Include performance metrics?
- **Yes**: Time per game is important for Phase 2 planning
- **How**: Time each matchup, divide by num_games
- **Why**: Helps set compute budget for learning experiments

## Success Criteria

This checkpoint is successful if it answers:

1. **Skill ceiling**: What's the best MC can do? (vs Random, vs Heuristic)
2. **Diminishing returns**: Where does scaling stop helping significantly?
3. **Game depth**: Does continued scaling improve play? (suggests strategic depth)
4. **Phase 2 target**: What win rate should learning agents aim for?
